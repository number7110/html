claude.ai(またはアプリ)を開き、下のプロンプトを送る。
この一文が、以降ずっと質問できる「隣のガイド」になる。

コピペ用プロンプト(チャットに送る)

Claude Code を初めて導入します。目標は、自社の資料(スライドや文書)を Claude Code で編集したり、HTML ページにしたりできるようになることです。私はエンジニアではなく、ターミナルにも慣れていません。30分ほどで終わる想定で、手順を一つずつ、専門用語をかみくだいて教えてください。一度にたくさん指示せず、1ステップずつ、私が「できた」と返してから次に進んでください。途中で詰まったら、このチャットで質問したり、画面のキャプチャを貼ったりするので、一緒に解決しながら進めてください。

この先もぜんぶ同じ——迷ったら聞く。エラーは画面を貼って聞く。

Step 2

ターミナルを開く

ターミナルは、コンピュータに文字で指示を出す画面。今日打つのはごく少数。

Mac

Command + Space → 「ターミナル」と入力 → Enter

Windows

スタートで「PowerShell」を検索 → 「Windows PowerShell」を開く

Step 3

Claude Code をインストール

下のコマンドをコピーして、ターミナルに貼り付けて Enter(推奨の方法。Node.js 等の準備は不要)。

MAC / LINUXcurl -fsSL https://claude.ai/install.sh | bash
WINDOWS (POWERSHELL)irm https://claude.ai/install.ps1 | iex
確認

ターミナルを一度閉じて開き直し、claude --version でバージョンが出れば成功。作業したいフォルダで claude と打つと、ブラウザでサインイン(Pro 以上のアカウント)。

詰まったら(例:「command not found」)——キャプチャと一緒にチャットへ

Claude Code のインストールで詰まっています。この下に貼る画面が出ています。まず新しいターミナルを開き直しましたが直りません。claude doctor の実行結果も必要なら指示してください。原因として何が考えられるかと、直し方を、初心者向けに一つずつ順番に教えてください。コマンドが必要な場合は、どこに何を貼り付ければよいかも具体的にお願いします。(この下に画面のキャプチャを貼る)

Step 4 — 今日の最難関

Drive につなぐ——鍵は"接続の許可証"

「つなぐ」といってもボタン一発ではなく、Google Cloud 側で認証情報を作る必要がある。手順は覚えなくて大丈夫——全体像だけ頭に入れて、画面操作はチャットに聞きながら。

① 素材を用意

手元の PowerPoint を1つ、Google スライドにアップロードしておく。

② Google Cloud で許可証を作る

プロジェクト作成 → Drive API 有効化 → OAuth 同意画面 → OAuth クライアントID(ウェブアプリ)を作成し、ID とシークレットを控える。

③ claude.ai に登録

設定 → コネクタ → カスタムコネクタを追加。MCP サーバーの URL と、控えた ID・シークレットを入力。

②で使う リダイレクトURIhttps://claude.ai/api/mcp/auth_callback
③で使う MCPサーバーURL(例)https://drivemcp.googleapis.com/mcp/v1

単純な「APIキー」ではなく、OAuth の ID とシークレットを使う——ここだけ注意。

Step 4 — コピペで進める

聞きながらつなぎ、指示するだけで編集

① 設定の手順をチャットに聞く

Claude Code から、自分の Google Drive のファイルを読み書きできるようにしたいです。次の設定を、初心者向けに一つずつ、画面のどこを押すかまで具体的に案内してください。(1) Google Cloud で新しいプロジェクトを作成する。(2) Google Drive API を有効化する。(3) OAuth 同意画面を設定する。(4) OAuth クライアントID(種類: ウェブアプリケーション)を作成し、リダイレクトURIに https://claude.ai/api/mcp/auth_callback を追加して、クライアントIDとクライアントシークレットを控える。(5) claude.ai の「設定 → コネクタ → カスタムコネクタを追加」で、MCP サーバーの URL と控えた ID・シークレットを登録する。1ステップずつ進めて、私が「できた」と返してから次に進んでください。エラーや分からない画面が出たらキャプチャを貼るので、一緒に直してください。

② つながったら、Code にスライドの編集を頼む 〈【 】は自分の内容に置き換えて送信〉

私の Google Drive にある「【スライドの名前】」というスライドを開いてください。開いたら、どのファイルを開いたかをまず教えてください。そのうえで、次の3点を直してください。(1) 表紙のタイトルを「【新しいタイトル】」に変更する。(2) 2枚目に「【追加したい内容】」をまとめたスライドを1枚追加する。(3) 全体の配色を、落ち着いたトーンに統一する。作業が終わったら、どこをどう変えたかを一覧で報告してください。

指示するだけで、資料がその場で書き換わっていく。
気に入らなければ「元に戻して」「もっと〇〇に」——会話で調整。

Step 5

そのスライドを HTML にする

仕上げは、いま編集したスライドをそのまま Web ページ(HTML)にしてもらう。
新しく用意するものは何もない。

コピペ用プロンプト(Code に送る) 〈【 】は自分の内容に置き換えて送信〉

先ほど編集した Google スライド「【スライドの名前】」を、そのまま1ページの HTML ファイルにしてください。要件は次のとおりです。(1) スライドの内容と順番は変えないこと。(2) 単一の index.html として保存し、スタイルもファイル1つで完結させること。(3) できあがったら、保存した場所と、ブラウザでの開き方を教えてください。表示が崩れる部分があれば、あとで「ここを直して」と頼むので、まずは全体を作ってください。

スライドが、そのままWeb ページになる。
気に入らないところは——もう分かるはず、会話で直せる。

Next 9

あとで試せる——持ち帰りメニュー9つ

①〜⑦は、右上のコピーで前提の確認→本編→振り返りまで一式の長文プロンプトが入る。今日の記憶が薄れていても、そのまま貼れば続きから始められる(【 】は置き換え可。例のままでもOK)。

① スキルを学ぶ

よく使うルールを「スキル(SKILL.md)」に保存→次回から自動で使われる。起動確認→仕組み→一緒に作成→保存→動作テストまで。

② スプレッドシートをつなぐ

資料で作った OAuth 許可証に Sheets API を追加。接続の生存確認→API有効化→OAuth確認→A1へ接続テスト→活用例まで。

③ ドキュメントもつなぐ

同じ許可証に Docs API を追加。API有効化→OAuth確認→文書末尾へ接続テスト→議事録整形などの活用例まで。

④ GAS を学ぶ

GASでスプレッドシートの定型作業を自動化。仕組み→コード→貼り付け→権限の警告の越え方→トリガー→エラー対応まで。

⑤ 作ったHTMLを公開する

index.html を Vercel でURLにして共有。ファイル探しから→デプロイ→更新→パスワード保護→取り下げ方まで。

⑥ Gmail・カレンダーで秘書に

標準コネクタをONにするだけ(OAuth不要)。権限の説明から→朝5分ブリーフィング→返信下書き→毎朝の定型プロンプトまで。

⑦ 決まった時間に自動で動かす

指示しなくても動く定期実行。方法の比較→推薦→設定→テスト→確認と止め方→暴走を防ぐ注意点まで。

⑧ 豆知識: モードの切り替え

Code の入力欄で Shift+Tab でモードが順に切り替わる。auto-accept = 確認なしで編集が進むオートモード / plan mode = まず計画だけ。困ったら Esc で停止。細かい許可は /permissions。

⑨ 豆知識: スキルの呼び出し方

スキルは頼みごとが当てはまると自動で使われる。明示的には /スキル名(例: /brand-rules)。名前で「〇〇のスキルを使って」でもOK。一覧は「使えるスキルを教えて」。

参考: docs.claude.com(セットアップ)・support.claude.com(Drive連携) ※最新は公式かチャットで確認。
